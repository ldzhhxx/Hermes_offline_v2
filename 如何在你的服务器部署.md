# 如何在你的服务器上部署 Hermes Offline v2

## 📦 文件说明

本次构建生成了以下文件供你在自己的服务器上部署：

1. **hermes-offline-image.tar** (2.3GB)
   - 完整的Docker镜像，包含所有依赖
   - 可以直接加载到任何Docker环境

2. **DEPLOYMENT_GUIDE.md**
   - 详细的部署指南
   - 包含多种部署方式
   - 安全配置建议
   - 故障排查方法

3. **quick-start.sh**
   - 一键启动脚本
   - 自动检测环境
   - 交互式配置

## 🚀 最快速的部署方法

### 第一步：传输文件到你的服务器

```bash
# 方法1: 使用scp传输镜像文件
scp hermes-offline-image.tar your-user@your-server:/path/to/destination/

# 方法2: 使用rsync传输（支持断点续传）
rsync -avz --progress hermes-offline-image.tar your-user@your-server:/path/to/destination/
```

### 第二步：在服务器上加载镜像

```bash
# SSH登录到服务器
ssh your-user@your-server

# 进入目标目录
cd /path/to/destination/

# 加载Docker镜像
docker load -i hermes-offline-image.tar

# 验证镜像已加载
docker images | grep hermes-offline
```

### 第三步：运行容器

#### 方式A：使用一键启动脚本（推荐新手）

```bash
# 如果你也上传了 quick-start.sh
chmod +x quick-start.sh
./quick-start.sh
```

#### 方式B：手动运行（更灵活）

```bash
# 创建数据目录
mkdir -p data workspace

# 启动容器
docker run -d \
  --name hermes-offline \
  -p 18789:18789 \
  -p 5000:5000 \
  -v $(pwd)/data:/home/hermes/.hermes \
  -v $(pwd)/workspace:/home/hermes/workspace \
  -e API_SERVER_KEY="your-secure-key-here" \
  -e HERMES_WEBUI_PASSWORD="your-password" \
  -e GATEWAY_ALLOW_ALL_USERS=true \
  --restart unless-stopped \
  hermes-offline:latest

# 查看日志
docker logs -f hermes-offline
```

### 第四步：访问服务

在浏览器中打开：
- **http://your-server-ip:18789** （WebUI界面）
- **http://your-server-ip:5000/health** （API健康检查）

## 🔒 防火墙配置

如果无法访问，需要开放端口：

### Ubuntu/Debian (UFW)
```bash
sudo ufw allow 18789/tcp
sudo ufw allow 5000/tcp
sudo ufw reload
```

### CentOS/RHEL (firewalld)
```bash
sudo firewall-cmd --add-port=18789/tcp --permanent
sudo firewall-cmd --add-port=5000/tcp --permanent
sudo firewall-cmd --reload
```

### 云服务器
在云服务商的控制台中配置安全组/防火墙规则：
- 开放入站规则
- 协议：TCP
- 端口：18789, 5000
- 来源：0.0.0.0/0 (或限制为你的IP)

## 🔐 安全建议

### 生产环境必做

1. **修改默认密码**
   ```bash
   -e HERMES_WEBUI_PASSWORD="your-strong-password"
   ```

2. **限制访问来源**
   - 在防火墙中只允许特定IP访问
   - 或使用VPN

3. **使用HTTPS**
   - 配置Nginx反向代理
   - 使用Let's Encrypt免费SSL证书

4. **定期备份数据**
   ```bash
   tar -czf hermes-backup-$(date +%Y%m%d).tar.gz data/ workspace/
   ```

## 📊 验证部署

### 检查服务状态
```bash
# 容器是否运行
docker ps | grep hermes-offline

# API健康检查
curl http://localhost:5000/health
# 应该返回: {"status": "ok", "platform": "hermes-agent"}

# WebUI是否响应
curl -I http://localhost:18789
# 应该返回: HTTP/1.1 200 OK
```

### 查看日志
```bash
# 实时日志
docker logs -f hermes-offline

# 最近100行
docker logs --tail 100 hermes-offline
```

## ❓ 常见问题

### Q: 镜像文件太大，传输很慢怎么办？
A: 
1. 使用rsync支持断点续传
2. 或者直接在服务器上clone仓库并构建镜像
3. 压缩后再传输：`gzip hermes-offline-image.tar`

### Q: 容器启动后无法访问？
A: 检查以下几点：
1. 容器是否正在运行：`docker ps`
2. 端口是否被占用：`netstat -tulpn | grep 18789`
3. 防火墙是否开放端口
4. 查看容器日志：`docker logs hermes-offline`

### Q: 如何更新到新版本？
A:
```bash
# 停止旧容器（数据在挂载卷中，不会丢失）
docker stop hermes-offline
docker rm hermes-offline

# 加载新镜像
docker load -i hermes-offline-image-new.tar

# 启动新容器（使用相同的挂载卷）
docker run -d \
  --name hermes-offline \
  -p 18789:18789 -p 5000:5000 \
  -v $(pwd)/data:/home/hermes/.hermes \
  -v $(pwd)/workspace:/home/hermes/workspace \
  --restart unless-stopped \
  hermes-offline:latest
```

### Q: 如何配置OpenAI/Anthropic API？
A: 创建 `.env` 文件或在启动时添加环境变量：
```bash
docker run -d \
  --name hermes-offline \
  -p 18789:18789 -p 5000:5000 \
  -v $(pwd)/data:/home/hermes/.hermes \
  -v $(pwd)/workspace:/home/hermes/workspace \
  -e OPENAI_API_KEY="sk-your-key" \
  -e ANTHROPIC_API_KEY="sk-ant-your-key" \
  hermes-offline:latest
```
